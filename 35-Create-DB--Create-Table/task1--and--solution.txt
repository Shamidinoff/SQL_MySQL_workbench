2. Добавьте в SQL-код к столбцам следующие ограничения: 

- ID - это табельный номер. Он уникален для каждого сотрудника и однозначно его идентифицирует (люди\персоны\сотрудники у нас строки - значит значения столбца ID однозначно идентифицируют строки). Поэтому сделайте его первичным ключом (PRIMARY KEY), а также автоматически генерируемым  AUTO_INCREMENT (при регистрации каждого нового сотрудника, т.е, появления его записи\строки в таблице, ему присваивается таб.номер +1 к предыдущему номеру).   Ну и вдобавок к целому числу надо дописать UNSIGNED - чтобы столбец не принимал отрицательные значения. 
- Name  - это ФИО человека\сотрудника. Пропущенным (null) оно быть не может (NOT NULL)  - т.е., надо хоть что-то написать. Ну и стоит поменять тип данных: хранить 65 Кб текста нам не надо - хватит 100 символов (VARCHAR (100)).
- EmplDate - дата начала сотрудничества (приема на работу). Не может быть позже даты основания компании 12 августа 2010 года, но мы пока ограничимся этой датой просто как значением по умолчанию (DEFAULT "2010-08-12")
- Salary  - не может быть отрицательным (UNSIGNED)
Запустите SQL, зайдите в настройки таблицы - и убедитесь, что все ограничения проставились на столбцах в БД. 

Подсказка: 

CREATE DATABASE test;
USE test;
CREATE TABLE Person
(ID INT UNSIGNED  PRIMARY KEY AUTO_INCREMENT,
Name VARCHAR(100) NOT NULL,
EmplDate DATE DEFAULT "2010-08-12",
Birth DATE,
Salary INT UNSIGNED)
